这一天的重点是：你不再只是“能训练一个图像分类模型”，而是开始理解**为什么卷积神经网络更适合图像**。

本文档分成两大部分：

1. 深度学习知识
2. 当前项目重要代码讲解

---

## 第一部分：深度学习知识

### 1. 为什么 MLP 对图片不够理想

第 2 天的 `MLP` 做法是：

```text
1 x 28 x 28 -> 784
```

也就是把整张图片直接展平成一个长向量。

这样做的优点是简单，但有一个明显问题：

- 它把图片的二维结构打散了
- 它不知道“相邻像素之间通常更相关”
- 它不知道边缘、纹理、局部形状这些图像特征

比如，一只鞋的鞋带区域和鞋底区域在图片中的空间关系是有意义的。
MLP 在展平后，并不会天然保留这种“局部邻近关系”。

---

### 2. CNN 为什么更适合图像

CNN 的核心思想是：

- 用小卷积核在图片上滑动
- 每次只看一个局部区域
- 学习边缘、纹理、局部图案
- 再逐层组合成更高级特征

你可以把 CNN 理解成：

- 第一层看到简单边缘
- 第二层看到更复杂的局部纹理
- 更后面的层看到局部形状组合
- 最后分类器根据这些高级特征判断类别

这比“把图片粗暴拉直后直接分类”更符合图像本身的结构。

---

### 3. 卷积层在做什么

卷积层最常见的写法是：

```python
nn.Conv2d(in_channels=1, out_channels=16, kernel_size=3, padding=1)
```

这行代码的含义：

- `in_channels=1`
  输入是灰度图，所以通道数为 1
- `out_channels=16`
  这一层会学出 16 个不同的特征图
- `kernel_size=3`
  每个卷积核大小是 `3 x 3`
- `padding=1`
  在边缘补 1 圈，让卷积后高宽保持不变

你可以把一个卷积核理解成一个“小探测器”：

- 有的探测器擅长发现横向边缘
- 有的擅长发现竖向边缘
- 有的擅长发现某种纹理模式

---

### 4. ReLU 在做什么

`ReLU` 是激活函数：

```python
nn.ReLU()
```

它的规则很简单：

- 大于 0 的数保留
- 小于等于 0 的数变成 0

它的作用不是“让模型更复杂”这么简单，而是：

- 给网络加入非线性能力
- 让多层叠加后不只是简单线性变换

没有激活函数，再深的网络也可能退化成一个大线性变换。

---

### 5. 池化层在做什么

我们的代码里使用了：

```python
nn.MaxPool2d(kernel_size=2)
```

它的作用是把特征图尺寸缩小一半。

例如：

```text
28 x 28 -> 14 x 14
14 x 14 -> 7 x 7
```

它的好处包括：

- 降低计算量
- 压缩特征图
- 保留最显著的局部响应

你可以把它理解成：在每个 `2 x 2` 小区域里，只保留最强信号。

---

### 6. CNN 中的张量 shape 怎么变化

在当前项目的 CNN 里，一张输入图片的 shape 变化如下：

```text
输入图片:               [batch, 1, 28, 28]
第一次卷积后:           [batch, 16, 28, 28]
第一次池化后:           [batch, 16, 14, 14]
第二次卷积后:           [batch, 32, 14, 14]
第二次池化后:           [batch, 32, 7, 7]
展平后:                 [batch, 32*7*7]
全连接隐藏层后:         [batch, 128]
输出 logits:            [batch, 10]
```

这个 shape 变化非常重要，因为它能帮你真正看懂 CNN。

---

### 7. 为什么最后还需要全连接层

卷积层负责提取特征，但最后我们还是要输出 10 个类别分数。

所以通常做法是：

1. 卷积层提取图像特征
2. 池化层压缩特征图
3. 展平
4. 全连接层把高级特征映射到类别 logits

也就是说：

- 前半部分：负责“看图”
- 后半部分：负责“做分类决策”

---

### 8. 为什么仍然使用 CrossEntropyLoss

虽然模型从 `MLP` 换成了 `CNN`，但任务本质没变：

- 还是 10 类单标签分类
- 每张图只属于一个类别

所以损失函数依然是：

```python
nn.CrossEntropyLoss()
```

这也说明一个很重要的工程事实：

- 模型结构可以变
- 只要任务形式不变，损失函数未必需要变

---

### 9. 为什么要和 MLP 做对比

第 3 天的价值不只是“会用 CNN”，而是你要知道：

**CNN 到底比 MLP 好在哪里。**

如果没有基线，你只会知道“CNN 能跑”。
有了 MLP 基线，你才会知道：

- CNN 的准确率通常更高
- CNN 更能利用图像局部结构
- CNN 不是“更复杂所以更好”，而是“更适合图像”

---

### 10. 这一天真正学会了什么

完成第 3 天后，你应该能回答这些问题：

- 为什么图像任务常用 CNN 而不是 MLP
- 卷积核为什么能提取局部特征
- 池化层为什么能降低尺寸
- 一张图进入 CNN 后 shape 如何变化
- 为什么 CNN 对图像更合适

## 第二部分：当前项目代码讲解

### 11. `model.py` 里的 `FashionMNISTCNN`

看 [model.py](D:\吕骐豪 工程文件\FashionMNIST\model.py)。

我们新增了：

```python
class FashionMNISTCNN(nn.Module):
```

这是第 3 天的核心模型。

#### 11.1 `self.features`

```python
self.features = nn.Sequential(
    nn.Conv2d(in_channels=1, out_channels=16, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(kernel_size=2),
    nn.Conv2d(in_channels=16, out_channels=32, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(kernel_size=2),
)
```

这一段负责“提取视觉特征”。

你可以把它看成两个卷积块：

第一块：
- 从原始灰度图提取 16 张基础特征图
- 再池化到 `14 x 14`

第二块：
- 在第一层提取的特征基础上继续提取更高级特征
- 通道数扩展到 32
- 再池化到 `7 x 7`

#### 11.2 `self.classifier`

```python
self.classifier = nn.Sequential(
    nn.Flatten(),
    nn.Linear(32 * 7 * 7, 128),
    nn.ReLU(),
    nn.Linear(128, num_classes),
)
```

这段负责“根据提取到的特征做分类”。

这里的 `32 * 7 * 7` 来自卷积部分输出的 shape。
如果你对这个数字能自己算出来，说明你已经开始真正理解 CNN 了。

#### 11.3 `forward`

```python
def forward(self, images):
    features = self.features(images)
    logits = self.classifier(features)
    return logits
```

非常值得注意的一点：

- `forward` 写得越清楚，你越容易理解数据流
- 这里明显分成“特征提取”和“分类”两步

### 12. `build_model()` 为什么有用

同样在 [model.py](D:\吕骐豪 工程文件\FashionMNIST\model.py) 里：

```python
def build_model(model_name: str, hidden_dim: int = 256) -> nn.Module:
```

它的作用是：

- 如果输入 `"mlp"`，就创建 `FashionMNISTMLP`
- 如果输入 `"cnn"`，就创建 `FashionMNISTCNN`

这样训练脚本就不用自己写很多 `if/else` 来手动构造模型。

这是一种很常见的工程写法，叫“工厂函数”。

---

### 13. `train.py` 里的 `TrainingConfig`

第 3 天里，配置项比第 2 天更多了。

例如：

- `output_root`
- `mlp_hidden_dim`
- `run_baseline_mlp`

它们的意义分别是：

- `output_root`：结果保存到哪里
- `mlp_hidden_dim`：MLP 基线隐藏层宽度
- `run_baseline_mlp`：是否先训练 MLP 再训练 CNN

这使得第 3 天的脚本不只是“能训练”，而是具备了“实验管理”的味道。

---

### 14. `train_single_model()`

这是第 3 天最重要的训练组织函数之一。

它负责：

1. 根据名字创建模型
2. 定义损失函数和优化器
3. 打印数据概览
4. 训练多个 epoch
5. 记录每个 epoch 的指标
6. 返回模型和历史记录

这里体现了一个很好的工程思想：

- 训练一个模型的流程是通用的
- MLP 和 CNN 只是模型不同
- 所以可以复用同一套训练逻辑

---

### 15. `save_history()` 和 `save_model_weights()`

这两个函数让你的项目开始有“实验结果沉淀”。

#### `save_history()`

把每一轮的：

- train_loss
- train_acc
- test_loss
- test_acc

保存成 JSON 文件。

这样你后面就可以：

- 画曲线
- 写报告
- 做实验比较

#### `save_model_weights()`

把训练后的模型参数保存成 `.pt` 文件。

这意味着：

- 你以后可以重新加载模型
- 不用每次都重新训练
- 可以把训练结果真正留住

---

### 16. `run_training()` 现在做了什么

第 3 天的 `run_training()` 不再只是训练一个模型。

它会：

1. 读取 FashionMNIST
2. 创建输出目录
3. 按顺序训练：
   - MLP 基线
   - CNN
4. 保存两个模型的 history 和权重
5. 生成总 summary
6. 打印对比结果

这已经很接近一个小型、完整的深度学习实验脚本了。
