## 第一部分：深度学习知识与实验分析思路

### 1. 什么是超参数

超参数不是训练中自动学出来的参数，而是你在训练开始前自己设定的值。

比如：

- 学习率 `learning_rate`
- 批大小 `batch_size`
- dropout 比例
- epoch 数

和它相对的，是模型参数：

- 卷积层权重
- 全连接层权重
- 偏置

这些模型参数会在训练中通过梯度下降不断更新。

所以要记住：

- 超参数：你手动设定
- 模型参数：模型自动学习

---

### 2. 为什么第 4 天要比较学习率

学习率决定每次参数更新的步子有多大。

如果学习率太小：

- loss 会下降得很慢
- 模型可能需要更多 epoch 才能学好

如果学习率太大：

- 参数更新可能“跨太大步”
- loss 可能震荡，甚至不收敛

所以学习率的核心问题是：

**更新要快，但不能乱。**

在当前实验里，我们比较了：

- `0.0003`
- `0.001`
- `0.003`

这能让你直接看到：

- 小学习率更稳，但可能慢
- 大学习率可能更快，但有风险

---

### 3. 为什么第 4 天要比较 batch size

`batch size` 决定每次更新时，用多少张图来估计梯度。

如果 batch 小：

- 每次梯度估计更“吵”
- 更新更频繁
- 有时泛化会不错

如果 batch 大：

- 梯度更平滑
- 每个 epoch 更新次数更少
- 内存占用更高

所以 batch size 不是“越大越好”，而是在：

- 训练稳定性
- 更新频率
- 硬件资源

之间找平衡。

在当前实验里，我们比较了：

- `32`
- `64`
- `128`

---

### 4. Dropout 是什么

Dropout 是经典正则化方法之一。

它在训练时会随机“关闭”一部分隐藏单元的输出。
例如 dropout=0.3 表示：

- 大约 30% 的隐藏激活会被临时置 0

它的作用是减少模型对某些固定特征组合的依赖，从而减轻过拟合。

可以把它理解成：

- 不允许模型总依赖同一小撮神经元
- 逼着模型学到更稳健、更分散的表示

Dropout 常见效果：

- 训练准确率可能下降一点
- 测试集表现可能更稳

---

### 5. 第 4 天为什么不只看 accuracy

很多初学者只看最终准确率，但真正做实验时，这远远不够。

你还应该看：

- `train_loss`
- `test_loss`
- `train_acc`
- `test_acc`
- 曲线是否平滑
- 训练集和测试集是否差距过大

比如：

- 训练集很高、测试集一般：可能过拟合
- 两边都低：可能模型不够强，或学习率不合适
- loss 波动很大：可能学习率过高

所以第 4 天的重点之一是：

**不要只看一个数字，要看整个训练过程。**

---

### 6. 错误样本为什么重要

错误样本分析是深度学习项目里很有价值的一步。

因为它回答的是：

- 模型到底错在什么图上
- 哪些类别容易混淆
- 错误是随机的，还是有规律的

比如 FashionMNIST 里常见混淆可能包括：

- `Shirt` 和 `T-shirt/top`
- `Coat` 和 `Pullover`
- `Sandal` 和 `Sneaker`

如果你只看总体准确率，你不知道模型为什么错。
但一看错误样本，就可能马上发现：

- 某两类本来长得就像
- 图像轮廓模糊时模型更容易错
- 某些样本甚至人眼也容易分不清

这就是为什么第 4 天开始要加入错误样本展示。

---

### 7. 什么是“实验设计意识”

第 4 天真正培养的是一种思维方式：

不要只是改一个参数然后说“感觉变好了”。
要学会：

1. 固定其他条件
2. 只改变一个变量
3. 记录结果
4. 比较差异
5. 给出解释

比如学习率实验：

- 模型结构不变
- batch size 不变
- dropout 不变
- 只改学习率

这样你才能说：

“性能差异主要来自学习率，而不是别的因素。”

这就是实验设计的基本意识。

## 第二部分：第 4 天代码讲解

### 8. `model.py` 中 CNN 为什么加入 dropout

在 [model.py](D:\吕骐豪 工程文件\FashionMNIST\model.py) 里，CNN 分类器现在是：

```python
self.classifier = nn.Sequential(
    nn.Flatten(),
    nn.Linear(32 * 7 * 7, 128),
    nn.ReLU(),
    nn.Dropout(p=dropout),
    nn.Linear(128, num_classes),
)
```

这里最关键的新层是：

```python
nn.Dropout(p=dropout)
```

它只在训练时起作用。
在评估阶段，PyTorch 会自动关闭 dropout。

你可以把这一步理解成：

- 训练时故意让模型“别太依赖固定的隐藏特征”
- 测试时再用完整网络做预测

---

### 9. `build_model()` 为什么也要改

同样在 [model.py](D:\吕骐豪 工程文件\FashionMNIST\model.py) 中：

```python
def build_model(model_name: str, hidden_dim: int = 256, dropout: float = 0.0)
```

这里多加了一个 `dropout` 参数，是为了让训练脚本可以动态创建：

- 无 dropout 的 CNN
- 有 dropout 的 CNN

这让实验脚本只需要改配置，而不需要改模型代码主体。

---

### 10. `TrainingConfig` 的作用

在 [train.py](D:\吕骐豪 工程文件\FashionMNIST\train.py) 里：

```python
class TrainingConfig:
```

它现在负责全局实验设置，例如：

- `epochs`
- `train_limit`
- `test_limit`
- `output_root`
- `error_example_count`

特别值得注意的是：

- `train_limit=12000`
- `test_limit=2000`

这是一个很现实的教学设计。
因为第 4 天要跑多个实验，如果每次都用完整数据集，在 CPU 上会很慢。
所以默认先用一个较小但足够分析趋势的子集。

等你需要更完整结果时，可以改成：

```python
train_limit=None
test_limit=None
```

---

### 11. `ExperimentSpec` 是什么

第 4 天最大的工程升级之一，是把每个实验写成一个结构化对象：

```python
class ExperimentSpec:
    name: str
    category: str
    learning_rate: float
    batch_size: int
    dropout: float
    epochs: int
    note: str
```

这表示每个实验都有自己完整的设定。

它带来的好处是：

- 不容易把实验配错
- 日后可以保存和复现
- 代码更清晰，不容易写出一堆零散的硬编码

---

### 12. `get_experiment_suite()` 在做什么

这个函数负责定义第 4 天要跑哪些实验。

目前包括：

- `baseline`
- `lr_low`
- `lr_high`
- `batch_32`
- `batch_128`
- `dropout_30`

它其实就是“实验计划表”。

这样你不需要手动一遍遍改代码再重跑，而是可以让脚本自己批量完成。

---

### 13. `make_fashion_mnist_dataloaders()` 为什么新增了 limit

在 [dataset.py](D:\吕骐豪 工程文件\FashionMNIST\dataset.py) 中，这个函数现在支持：

- `train_limit`
- `test_limit`
- `subset_seed`

作用是：

- 从完整数据集中随机选一个固定子集
- 保证不同实验用的是同一批数据
- 这样实验比较更公平

这里很关键的一点是：

**你比较学习率时，不应该因为数据子集不同而影响结果。**

所以用了固定随机种子。

---

### 14. `train_experiment()` 是第 4 天最核心的函数

它的职责是：

1. 根据实验配置创建 DataLoader
2. 创建对应的 CNN
3. 定义损失函数和优化器
4. 训练多个 epoch
5. 记录每轮指标
6. 收集错误样本
7. 保存图、JSON、权重

也就是说，单个实验从训练到分析产物落盘，都是它负责的。

这已经很接近一个真正小型实验平台的雏形。

---

### 15. 为什么要保存 `history.json`

每一轮的：

- train_loss
- train_acc
- test_loss
- test_acc

都会保存到 `history.json`。

这有两个价值：

1. 你可以回头画曲线
2. 你可以写实验报告时直接引用结果

这比只在终端里看一眼结果要有价值得多。

---

### 16. `save_history_plot()` 为什么重要

这个函数会自动把训练历史画成两张曲线：

- loss 曲线
- accuracy 曲线

这一步很重要，因为很多训练现象用文字很难判断，但一看曲线就很直观。

例如：

- 曲线平滑下降：训练较稳定
- 曲线剧烈震荡：学习率可能偏大
- 训练集上升很快，测试集不上升：可能过拟合

---

### 17. `collect_misclassified_examples()` 在做什么

这是错误分析的核心函数。

它会：

1. 用训练好的模型跑测试集
2. 找出预测错误的样本
3. 保存这些样本的：
   - 图片
   - 真实标签
   - 预测标签
   - 预测置信度

这让后面画错误样本图成为可能。

---

### 18. `save_error_samples_plot()` 为什么有价值

这个函数会把错误样本画成图片网格，方便你直接看。

图里每张样本都会标出：

- 真值类别
- 预测类别
- 预测置信度

这一步对报告和学习都很有帮助，因为你终于可以把“模型错了”变成“模型在什么类型的样本上错了”。

---

### 19. `save_experiment_comparison_plot()` 在做什么

这个函数会把不同实验的测试准确率画成柱状图。

这是第 4 天最直接的总结图之一。

你会很直观地看到：

- 哪个实验最好
- 哪个实验最差
- 学习率、batch size、dropout 分别带来了什么变化

---

### 20. `run_training()` 现在已经不是单纯训练函数了

在第 4 天，`run_training()` 更像一个“实验总控函数”。

它会：

1. 读取配置
2. 生成实验清单
3. 逐个执行实验
4. 收集每个实验结果
5. 选出最佳实验
6. 保存总体 summary
7. 打印最终对比表

这一步标志着你的项目正在从“单模型训练脚本”过渡到“可管理的实验工程”。

## 第三部分：你应该怎么使用这些结果

### 21. 你运行完以后先看什么

建议顺序是：

1. 先看终端里的总体对比表
2. 再看 `day4_summary.json`
3. 再看每个实验子目录里的 `curves.png`
4. 最后看 `misclassified_examples.png`

这是一个非常合理的分析顺序：

- 先看总体谁最好
- 再看过程曲线
- 最后看模型究竟错在哪

---

### 22. 写实验分析时可以怎么写

你可以按这样的结构写：

1. 基线设置是什么
2. 学习率实验结果如何
3. batch size 实验结果如何
4. dropout 实验结果如何
5. 错误样本显示模型常见混淆类别是什么
6. 最终选用哪个设置，为什么

这已经是非常标准、很像课程项目报告的写法了。
